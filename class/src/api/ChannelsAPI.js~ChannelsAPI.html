    public create(params: object): Promise<Channel> source

    POST /channels/create

    Create a channel and place it into a stasis application. This is similiar to originate except the created channel is not dialed. This allows for an application writer to create a channel and perform manipulations to it prior to performing dial on the channel.

    API available since Asterisk 14.0

    Params:

    NameTypeAttributeDescription
    params object
    params.endpoint string

    The endpoint for channel communication.
    params.app string

    The stasis application to place the channel into.
    params.appArgs string
    • optional

    The application arguments to pass to the stasis application.
    params.channelId string
    • optional

    The unique id to assign the channel on creation.
    params.otherChannelId string
    • optional

    The unique id to assign the second channel when using local channels.
    params.originator string
    • optional

    The unique ID of the calling channel.
    params.formats string | Array<string>
    • optional

    The format name capability list to use if originator is not specified. For example, ['ulaw', 'slin16']. Format names can be found by executing the core show codecs command in the Asterisk console.

    Return:

    Promise<Channel>

    Resolves with the newly created channel. Rejects when invalid params are passed (status 400).

    public originate(params: object): Promise<Channel> source

    POST /channels

    Create a new channel (originate). The new channel is created immediately and a snapshot of it is returned. If a stasis application is provided, that application will be automatically subscribed to the originated channel for further events and updates.

    Params:

    NameTypeAttributeDescription
    params object
    params.endpoint string

    The endpoint to call
    params.extension string
    • optional

    The extension to dial after the endpoint answers. Mutually exclusive with app.
    params.context string
    • optional

    The context to dial after the endpoint answers. If omitted, uses 'default'. Mutually exclusive with app.
    params.priority number
    • optional

    The priority to dial after the endpoint answers. If omitted, uses 1. Mutually exclusive with app.
    params.label string
    • optional

    The label to dial after the endpoint answers. Will supersede params.priority if both are provided. Mutually exclusive with app. Param available since Asterisk 13.2
    params.app string
    • optional

    The application that is subscribed to the originated channel. When the channel is answered, it will be passed to this stasis application. Mutually exclusive with context, extension, priority, and label.
    params.appArgs string
    • optional

    The application arguments to pass to the stasis application provided by app. Mutually exclusive with context, extension, priority, and label.
    params.callerId string
    • optional

    CallerID to use when dialing the endpoint or extension.
    params.timeout number
    • optional
    • default: 30

    The timeout in seconds before giving up dialing, or -1 for no timeout.
    params.channelId string
    • optional

    The unique id to assign the channel on creation. Param available since Asterisk 12.2
    params.otherChannelId string
    • optional

    The unique id to assign the second channel when using local channels.
    params.originator string
    • optional

    The unique id of the channel which is originating this one. Param available since Asterisk 13.2
    params.formats string | Array<string>
    • optional

    The format name capability list to use if originator is not specified. For example, ['ulaw', 'slin16']. Format names can be found by executing the core show codecs command in the Asterisk console. Param available since Asterisk 13.10
    params.variables object
    • optional

    An object of key/value pairs that should be set on the channel on creation. Param available since Asterisk 12.1

    Return:

    Promise<Channel>

    Resolves with the newly created channel. Rejects when invalid params are passed (status 400).

    public play(params: object): Promise<Playback> source

    POST /channels/{channelId}/play

    Start playback of media. The media URI may be any of a number of URIs. sound:, recording:, number:, digits:, characters:, and tone: URIs are supported. This operation creates a playback resource that can be used to control the playback of media (pause, rewind, fast forward, etc.)

    'tone:' playback URI added in Asterisk 12.3

    Params:

    NameTypeAttributeDescription
    params object
    params.channelId string

    the id of the channel to play the media to.
    params.media string | Array<string>

    The media's URI to play. Allows multiple media to be passed since Asterisk 14.0
    params.lang string
    • optional

    For sounds, the language for the sound.
    params.offsetms number
    • optional
    • default: 0

    The number of milliseconds to skip before playing the media URI. Allowed range: 0+
    params.skipms number
    • optional
    • default: 3000

    The number of milliseconds to skip for forward/reverse operations. Allowed range: 0+
    params.playbackId string
    • optional

    The identifier of the playback that is started. Param available since Asterisk 12.2

    Return:

    Promise<Playback>

    Resolves with the details of the started playback.

    public record(params: object): Promise<LiveRecording> source

    POST /channels/{channelId}/record

    Start a recording. Record audio from the channel. Note that this will not capture audio send to the channel. The bridge itself has a record feature if that's what you want.

    Params:

    NameTypeAttributeDescription
    params object
    params.channelId string

    The id of the channel to record.
    params.name string

    The recording's filename.
    params.format string

    The format to record the audio in.
    params.maxDurationSeconds number
    • optional
    • default: 0

    The maximum duration of the recording, in seconds. 0 for no limit. Allowed range: 0+
    params.maxSilenceSeconds number
    • optional
    • default: 0

    The maximum duration of silence, in seconds. o for no limit. Allowed range: 0+
    params.ifExists string
    • optional
    • default: fail

    The action to take if a recording with the same name already exists. Allowed values: 'fail', 'overwrite', or 'append'.
    params.beep boolean
    • optional
    • default: true

    Play beep when recording begins.
    params.terminateOn string
    • optional
    • default: none

    DTMF input to terminate recording. Allowed values: 'none', 'any', '*', or '#'.

    Return:

    Promise<LiveRecording>

    Resolves with the details of the ongoing recording. Rejects if invalid parameters are provided (status 400), if the channel is not found (status 404); if the channel is not in a stasis application (status 409); if the channel is currently bridged with other channels (status 409); if a recording of the same name already exists on the system and cannot be overwritten because it is in progress or ifExists was set to 'fail' (status 409); or if the format specified is unknown on the system (status 422).
